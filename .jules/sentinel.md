## 2024-04-28 - [CRITICAL] Untrusted Input in Log Parsing Leading to Injection Vulnerabilities
**Vulnerability:** Extracted inputs from local log files (like game titles, usernames, IP addresses, and PIDs) were being used directly in sensitive operations. For example, unvalidated IP addresses were used as arguments for the shell command `ping` via Swift's `Process()`, and interpolated into `ip-api.com` URLs, exposing the application to command/argument injection and Server-Side Request Forgery (SSRF) respectively. The `placeId` and `pid` fields were similarly exposed in other contexts.
**Learning:** Application log files are a form of untrusted input. Malicious actors could manipulate the contents of these log files (e.g., through specifically crafted usernames or game actions that are logged locally) to inject commands or trigger unauthorized network requests. We must never assume log contents are benign.
**Prevention:** All data extracted from log parsing must be strictly validated against an expected format (e.g., `^[a-fA-F0-9.:]+$` for IPs, `^[0-9]+$` for PIDs) before being used in commands, API URLs, or any other sensitive context. When possible, use native macOS APIs over external binaries via `Process`.
