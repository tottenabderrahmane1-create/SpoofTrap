## 2025-01-20 - Command Injection via Path Interpolation in Shell Execution
**Vulnerability:** The `relaunchApp` function in `UpdateChecker.swift` used string interpolation to insert `Bundle.main.bundleURL.path` into a `/bin/sh -c` shell command (`open "\\(appPath)"`). This created a local command injection vulnerability. If the application bundle was maliciously renamed by a user or another process (e.g., to `SpoofTrap"; rm -rf /; ".app`), the embedded shell command would execute the injected payload when the app attempted to restart after an update.
**Learning:** Using `Process()` with `/bin/sh -c` and string interpolation for any dynamic input, even seemingly safe ones like application paths, is extremely dangerous and a classic vector for command injection. macOS application paths are entirely user-controllable.
**Prevention:** Never use string interpolation to build shell commands. Prioritize native macOS APIs over calling external binaries via `Process`. In this case, use `NSWorkspace.shared.openApplication(at:configuration:completionHandler:)` with `createsNewApplicationInstance = true` for relaunching/opening apps safely without relying on a shell.
