## 2024-05-24 - Command Injection and SSRF via Log Parsing
**Vulnerability:** The application parses variables like IP addresses and PIDs from external application logs and uses them directly in shell commands (`/bin/ps`, `/sbin/ping`) and URLs (`http://ip-api.com/json/...`, `https://games.roblox.com/...`). Log contents are untrusted external inputs. Malicious log entries could trigger command injection or Server-Side Request Forgery.
**Learning:** Even though the source of the log is a local application, its content is not trusted. The environment can be tampered with, or the application might log user-controlled input (like usernames that mimic log structures). Unsanitized strings extracted from logs should never be trusted as command arguments or URL components.
**Prevention:** Always apply strict format validation (e.g., regex `^[0-9]+$` for PIDs, `^[a-fA-F0-9.:]+$` for IPs) to variables parsed from logs or untrusted sources before passing them to sensitive functions like `Process()` arguments or `URL(string:)`.
