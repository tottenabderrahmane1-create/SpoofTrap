## 2024-04-17 - Unvalidated Log Parsing Input Used in Sensitive Operations
**Vulnerability:** Extracted variables from log files (e.g., IPs and PlaceIDs) were directly passed to system shell commands via `Process()` and external API requests via `URL(string:)` without structural validation. This created Server-Side Request Forgery (SSRF) and Command/Argument Injection risks if a malicious log entry was injected or misinterpreted.
**Learning:** Even internal logs generated by trusted applications (like Roblox) can be an attack vector if the parsing logic extracts values blindly. Applications must treat all parsed log input as untrusted.
**Prevention:** Strictly validate all parsed log data against expected regex patterns (e.g., `^[a-fA-F0-9.:]+$` for IPs and `^[0-9]+$` for IDs) before using them in execution contexts or URL constructions.
