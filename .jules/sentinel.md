## 2024-05-18 - Fix Command Injection via String Interpolation in App Relaunch
**Vulnerability:** Found a command injection vulnerability in `Sources/UpdateChecker.swift` where `Bundle.main.bundleURL.path` was directly interpolated into a `/bin/sh -c` argument (`task.arguments = ["-c", "sleep 1 && open \"\(appPath)\""]`). If an attacker managed to trick the user into placing the app bundle in a directory with shell metacharacters or backticks in the name, arbitrary commands could be executed upon updating/relaunching the app.
**Learning:** Even internal or seemingly "safe" string values like an app's path can become a vector for command injection when interpolated directly into a shell string executed by `Process()` (specifically via `/bin/sh -c`).
**Prevention:** Always use positional arguments when invoking scripts via `Process()` with `sh -c`. Pass the user-controlled or variable data as separate elements in the `arguments` array (e.g., `["-c", "sleep 1 && open \"$0\"", appPath]`) so the shell treats it as a parameter, not as an executable string.
