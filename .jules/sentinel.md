## 2024-05-02 - Fix local command injection via bundle path in app relaunch
**Vulnerability:** The `UpdateChecker.swift`'s `relaunchApp` function had a local command injection vulnerability. `Bundle.main.bundleURL.path` was directly interpolated into a shell script string (`["-c", "sleep 1 && open \"\(appPath)\""]`). If an attacker renames the `SpoofTrap.app` bundle to include bash commands (e.g., `SpoofTrap"; touch /tmp/pwned; ".app`), running the app and triggering an update/relaunch would execute the payload.
**Learning:** Even internal properties like `Bundle.main.bundleURL.path` should be treated as untrusted input when passed to a shell. The system `open` command can be hijacked if the shell script executing it doesn't safely escape parameters.
**Prevention:** Always pass user-controllable or potentially variable paths as separate arguments to `/bin/sh` scripts (e.g., passing the path as a trailing argument and referencing it as `"$0"` within the script) rather than using string interpolation.
