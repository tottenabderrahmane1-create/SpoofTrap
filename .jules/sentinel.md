## 2024-04-18 - Fix Argument Injection and SSRF Risks in Log Watcher
**Vulnerability:** User inputs (like IPs, PIDs, and PlaceIDs) extracted from log files via Regex were passed directly into `Process()` arguments and unescaped URL interpolations without structural validation. While they were filtered through regexes matching specific formats during parsing, relying on loosely-parsed log strings in downstream shell commands and external URLs poses command injection and SSRF risks if the log parser fails or if an attacker can inject malicious strings into the log format.
**Learning:** Even if data is extracted via seemingly safe regular expressions from a log file, it must be explicitly re-validated against strict allowed patterns (e.g., `^[0-9]+$` for IDs/PIDs, `^[a-fA-F0-9.:]+$` for IPs) right before usage in sensitive contexts like `Process()` or network requests.
**Prevention:** Always validate external inputs using strict character sets or structural parsers (like `URLComponents`) before interpolating them into shell commands or URLs, and use parameterization wherever possible.
