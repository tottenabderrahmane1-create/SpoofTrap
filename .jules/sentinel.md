## 2024-05-24 - Missing Input Validation on Log Parsed Data
**Vulnerability:** The `RobloxLogWatcher` class extracted values (like process IDs, IPs, and Place IDs) directly from untrusted local log files using basic string manipulation and passed them unvalidated to sensitive sinks. This included shell command arguments via `Process()` (`/bin/ps`, `/sbin/ping`) and network requests via `URLSession`, exposing the application to command injection and Server-Side Request Forgery (SSRF) if the log file was manipulated.
**Learning:** Even local files (like application logs) must be treated as untrusted input sources. Values extracted from these files can be manipulated by malicious actors to inject commands or trigger unauthorized network requests.
**Prevention:** Always apply strict format validation (e.g., using regular expressions like `^[0-9]+$` for IDs or `^[a-fA-F0-9.:]+$` for IPs) to any data extracted from logs or other external files before using them in shell commands, SQL queries, or network requests.
