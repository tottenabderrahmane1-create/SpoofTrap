## 2024-05-14 - Prevent Command and Argument Injection in Log Parsing
**Vulnerability:** External input extracted from log files (e.g., IP addresses, Process IDs) was passed directly into `URL` strings (causing SSRF risks) and `Process` arguments (causing command/argument injection risks) without explicit format validation.
**Learning:** Even when inputs are extracted using string scanning (which might seem safe), they can be manipulated if the source log file is poisoned. Any data derived from an external source must be strictly validated against an expected format (e.g., regex `^[a-fA-F0-9.:]+$` for IPs or `^[0-9]+$` for PIDs) before use in sensitive operations like `Process()` arguments or URLs.
**Prevention:** Always implement explicit input validation, such as the `isValidIP` helper or strict regex checks, for any variable passed into shell commands, subprocess arguments, or external URLs, even if the variable originated from parsing an assumed-safe local file.
